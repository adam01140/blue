# Password Protection System Documentation
# This document explains how the password protection lock works with auth.js
# Designed to be detailed enough for an AI agent to replicate on any webpage

## OVERVIEW
This system provides site-wide password protection using:
- Server-side password verification (secure)
- Client-side token-based authentication (localStorage)
- Automatic page protection with redirects
- Session management with expiration (30 days default)

## ARCHITECTURE

### Components:
1. **Server Endpoint** (`/api/verify-password`) - Verifies passwords server-side
2. **Client Script** (`auth.js`) - Protects pages and manages authentication state
3. **Login Page** (`login.html`) - User interface for password entry
4. **LocalStorage** - Stores authentication token and expiry timestamp

---

## STEP-BY-STEP IMPLEMENTATION

### 1. SERVER-SIDE SETUP (Backend/API)

#### Create a password verification endpoint:

```javascript
// In your server.js (or equivalent backend file)
app.post('/api/verify-password', (req, res) => {
  try {
    const { password } = req.body;
    
    if (!password) {
      return res.status(400).json({ 
        success: false, 
        error: 'Password is required' 
      });
    }

    // Compare with your stored password (from environment variable)
    // In production, use environment variables: process.env.ADMIN_PASSWORD
    const ADMIN_PASSWORD = 'your-secure-password-here'; // Or from env
    
    if (password === ADMIN_PASSWORD) {
      res.json({ 
        success: true, 
        message: 'Authentication successful' 
      });
    } else {
      res.status(401).json({ 
        success: false, 
        error: 'Invalid password' 
      });
    }

  } catch (error) {
    console.error('Password verification error:', error);
    res.status(500).json({ 
      success: false, 
      error: 'Server error during authentication' 
    });
  }
});
```

**Important Notes:**
- Store the actual password in environment variables (e.g., `.env` file with `ADMIN_PASSWORD=yourpassword`)
- The endpoint returns `{ success: true/false }` - this format is critical
- Use POST method to avoid exposing passwords in URL
- Always use HTTPS in production

### 2. CLIENT-SIDE AUTHENTICATION SCRIPT (auth.js)

Create a file called `auth.js` with the following structure:

```javascript
// Authentication System
// This script protects all pages and only needs to be included once

(function() {
    'use strict';
    
    // ============================================
    // CONFIGURATION
    // ============================================
    const AUTH_TOKEN_KEY = 'formstar_auth_token'; // LocalStorage key for token
    const AUTH_EXPIRY_KEY = 'formstar_auth_expiry'; // LocalStorage key for expiry
    const SESSION_DURATION = 30 * 24 * 60 * 60 * 1000; // 30 days in milliseconds
    const VERIFY_PASSWORD_ENDPOINT = '/api/verify-password'; // Your API endpoint
    
    // Check if we're already on the login page (to prevent redirect loops)
    const currentPage = window.location.pathname.split('/').pop();
    const isLoginPage = currentPage === 'login.html' || currentPage.endsWith('login.html');
    
    // ============================================
    // PATH CALCULATION FUNCTIONS
    // ============================================
    
    // Calculate relative path to login page based on current directory depth
    function getLoginPagePath() {
        const path = window.location.pathname;
        // Split path and filter out empty strings and the HTML file
        const parts = path.split('/').filter(p => p && !p.endsWith('.html'));
        // Count directories (depth from root)
        const depth = Math.max(0, parts.length);
        // Build relative path:
        // - If depth is 0, we're at root: 'login.html'
        // - If depth is 1, we're in a subfolder: '../login.html'
        // - If depth is 2, we're 2 levels deep: '../../login.html'
        return depth > 0 ? '../'.repeat(depth) + 'login.html' : 'login.html';
    }
    
    // Get redirect URL (for after successful login)
    function getRedirectUrl() {
        // First, check URL query parameter for 'redirect'
        const params = new URLSearchParams(window.location.search);
        const redirect = params.get('redirect');
        if (redirect) {
            return redirect;
        }
        // Default to index.html (adjust path based on current location)
        const path = window.location.pathname;
        const parts = path.split('/').filter(p => p && !p.endsWith('.html'));
        const depth = Math.max(0, parts.length);
        return depth > 0 ? '../'.repeat(depth) + 'index.html' : 'index.html';
    }
    
    // ============================================
    // AUTHENTICATION STATE FUNCTIONS
    // ============================================
    
    // Check if user is currently authenticated
    function isAuthenticated() {
        const token = localStorage.getItem(AUTH_TOKEN_KEY);
        const expiry = localStorage.getItem(AUTH_EXPIRY_KEY);
        
        if (!token || !expiry) {
            return false;
        }
        
        // Check if session has expired
        const now = new Date().getTime();
        if (now > parseInt(expiry)) {
            // Session expired, clear storage
            localStorage.removeItem(AUTH_TOKEN_KEY);
            localStorage.removeItem(AUTH_EXPIRY_KEY);
            return false;
        }
        
        return true;
    }
    
    // Set authentication after successful password verification
    function setAuthenticated() {
        const now = new Date().getTime();
        const expiry = now + SESSION_DURATION;
        
        // Create a simple token (base64 encoded timestamp)
        // In a production app, this could be a JWT from the server
        const token = btoa('authenticated_' + now);
        
        localStorage.setItem(AUTH_TOKEN_KEY, token);
        localStorage.setItem(AUTH_EXPIRY_KEY, expiry.toString());
    }
    
    // Clear authentication (for logout)
    function clearAuthentication() {
        localStorage.removeItem(AUTH_TOKEN_KEY);
        localStorage.removeItem(AUTH_EXPIRY_KEY);
    }
    
    // ============================================
    // REDIRECTION FUNCTIONS
    // ============================================
    
    // Redirect to login page with current URL as redirect parameter
    function redirectToLogin() {
        const currentUrl = window.location.pathname + window.location.search;
        const loginPagePath = getLoginPagePath();
        const loginUrl = loginPagePath + '?redirect=' + encodeURIComponent(currentUrl);
        // Use replace instead of href to prevent back button from accessing protected page
        window.location.replace(loginUrl);
    }
    
    // ============================================
    // SERVER COMMUNICATION
    // ============================================
    
    // Verify password with server (async function)
    async function verifyPassword(password) {
        try {
            const response = await fetch(VERIFY_PASSWORD_ENDPOINT, {
                method: 'POST',
                headers: {
                    'Content-Type': 'application/json'
                },
                body: JSON.stringify({ password: password.trim() })
            });
            
            if (!response.ok) {
                console.error('Server response not OK:', response.status, response.statusText);
                const errorData = await response.json().catch(() => ({}));
                console.error('Error data:', errorData);
                return false;
            }
            
            const data = await response.json();
            // Check if server returned success: true
            return data.success === true;
        } catch (error) {
            console.error('Password verification error:', error);
            return false;
        }
    }
    
    // ============================================
    // EXPOSE FUNCTIONS GLOBALLY
    // ============================================
    
    // Expose authentication functions for login page
    // These must be available even on the login page itself
    window.auth = {
        checkPassword: verifyPassword, // Server-side password verification
        setAuthenticated: setAuthenticated,
        getRedirectUrl: getRedirectUrl,
        clearAuthentication: clearAuthentication
    };
    
    // Expose logout function globally
    window.logout = function() {
        clearAuthentication();
        window.location.href = getLoginPagePath();
    };
    
    // ============================================
    // MAIN PROTECTION LOGIC
    // ============================================
    
    // Only check authentication if NOT on login page (prevents redirect loop)
    if (!isLoginPage) {
        // Check authentication immediately (runs as soon as script loads)
        // This runs synchronously when the script is included in <head>
        // Must check before any page content loads
        try {
            if (!isAuthenticated()) {
                // Redirect immediately - this will stop page rendering
                redirectToLogin();
            }
        } catch (e) {
            // If there's any error, still redirect to login for security
            console.error('Auth check error:', e);
            try {
                window.location.replace(getLoginPagePath());
            } catch (e2) {
                // Last resort - try href
                window.location.href = getLoginPagePath();
            }
        }
    }
})();
```

**Key Implementation Points:**
1. **IIFE (Immediately Invoked Function Expression)** - Wraps code to avoid global namespace pollution
2. **Synchronous Check** - Runs immediately when script loads, before page renders
3. **Path Calculation** - Dynamically calculates relative paths based on directory depth
4. **LocalStorage Storage** - Stores token and expiry timestamp
5. **Expiry Checking** - Validates session hasn't expired before allowing access
6. **Server Communication** - Uses async/await to verify password with backend

### 3. LOGIN PAGE (login.html)

Create a login page with this structure:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Login</title>
    <!-- Your styles here -->
</head>
<body>
    <div class="login-container">
        <h1>Login</h1>
        <p>Please enter the password to continue</p>
        
        <form class="login-form" id="loginForm">
            <div class="form-group">
                <label for="password">Password</label>
                <input 
                    type="password" 
                    id="password" 
                    name="password" 
                    placeholder="Enter password" 
                    required 
                    autofocus
                    autocomplete="current-password"
                >
            </div>
            
            <div class="error-message" id="errorMessage"></div>
            
            <button type="submit" class="login-button" id="loginButton">
                Login
            </button>
            
            <div class="loading" id="loading" style="display: none;">
                <div class="spinner"></div>
            </div>
        </form>
    </div>
    
    <!-- IMPORTANT: Include auth.js BEFORE your login script -->
    <script src="auth.js"></script>
    
    <script>
        document.addEventListener('DOMContentLoaded', function() {
            const form = document.getElementById('loginForm');
            const passwordInput = document.getElementById('password');
            const errorMessage = document.getElementById('errorMessage');
            const loginButton = document.getElementById('loginButton');
            const loading = document.getElementById('loading');
            
            // Check if already authenticated (user might have navigated here directly)
            if (window.auth && typeof window.auth.checkPassword === 'function') {
                const token = localStorage.getItem('formstar_auth_token');
                const expiry = localStorage.getItem('formstar_auth_expiry');
                if (token && expiry && new Date().getTime() < parseInt(expiry)) {
                    // Already authenticated, redirect to original page or home
                    const redirectUrl = window.auth.getRedirectUrl() || '../index.html';
                    window.location.href = redirectUrl;
                    return;
                }
            }
            
            // Handle form submission
            form.addEventListener('submit', function(e) {
                e.preventDefault();
                
                const password = passwordInput.value.trim();
                
                // Clear previous error
                errorMessage.classList.remove('show');
                errorMessage.textContent = '';
                
                // Show loading state
                loginButton.disabled = true;
                loading.style.display = 'block';
                
                // Check password with server (async)
                if (window.auth && window.auth.checkPassword) {
                    window.auth.checkPassword(password).then(function(isValid) {
                        if (isValid) {
                            // Password correct - set authentication
                            window.auth.setAuthenticated();
                            
                            // Get redirect URL (from query parameter or default)
                            const redirectUrl = window.auth.getRedirectUrl() || '../index.html';
                            
                            // Redirect to original page or home
                            window.location.href = redirectUrl;
                        } else {
                            // Password incorrect
                            errorMessage.textContent = 'Incorrect password. Please try again.';
                            errorMessage.classList.add('show');
                            passwordInput.value = '';
                            passwordInput.focus();
                            loginButton.disabled = false;
                            loading.style.display = 'none';
                        }
                    }).catch(function(error) {
                        console.error('Password verification error:', error);
                        errorMessage.textContent = 'Error verifying password. Please try again.';
                        errorMessage.classList.add('show');
                        loginButton.disabled = false;
                        loading.style.display = 'none';
                    });
                } else {
                    // Fallback if auth object not available
                    errorMessage.textContent = 'Authentication system not available. Please refresh the page.';
                    errorMessage.classList.add('show');
                    loginButton.disabled = false;
                    loading.style.display = 'none';
                }
            });
            
            // Allow Enter key to submit
            passwordInput.addEventListener('keypress', function(e) {
                if (e.key === 'Enter') {
                    form.dispatchEvent(new Event('submit'));
                }
            });
        });
    </script>
</body>
</html>
```

**Key Features:**
- Form validation (required password field)
- Error message display
- Loading state during verification
- Auto-redirect if already authenticated
- Preserves redirect URL from query parameter

### 4. PROTECTING PAGES

To protect any HTML page, simply include `auth.js` in the `<head>` section:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Protected Page</title>
    
    <!-- IMPORTANT: Include auth.js in <head> to check auth before page loads -->
    <script src="auth.js"></script>
</head>
<body>
    <!-- Your page content here -->
    
    <!-- Optional: Add logout button -->
    <button onclick="window.logout()">Logout</button>
</body>
</html>
```

**For nested directories**, adjust the path:
- Root level: `<script src="auth.js"></script>`
- One level deep: `<script src="../auth.js"></script>`
- Two levels deep: `<script src="../../auth.js"></script>`

The script automatically calculates correct paths for redirects.

## SECURITY CONSIDERATIONS

### ✅ What This System Provides:
1. **Server-side verification** - Password is never stored in client code
2. **Session management** - Tokens expire after set duration
3. **Immediate protection** - Script runs before page content loads
4. **Redirect preservation** - User returns to intended page after login

### ⚠️ Limitations & Recommendations:

1. **LocalStorage Tokens**: The tokens are stored client-side and can be manually set. For stronger security:
   - Use JWT tokens from server
   - Implement token refresh mechanism
   - Add server-side session validation

2. **Password Storage**: Currently uses environment variables (good). For production:
   - Use password hashing (bcrypt, argon2)
   - Implement rate limiting on login attempts
   - Add CSRF protection

3. **HTTPS**: Always use HTTPS in production to prevent password interception

4. **Session Duration**: 30 days is long - consider shorter sessions for sensitive content

5. **LocalStorage Vulnerabilities**: 
   - Vulnerable to XSS attacks
   - Consider httpOnly cookies for token storage
   - Implement Content Security Policy (CSP)

## CUSTOMIZATION GUIDE

```javascript
// In auth.js, modify:

```

### Change LocalStorage Keys:
```javascript
// In auth.js, modify:
const AUTH_TOKEN_KEY = 'your_app_auth_token';
const AUTH_EXPIRY_KEY = 'your_app_auth_expiry';
```

```javascript
// In auth.js, modify:
const VERIFY_PASSWORD_ENDPOINT = '/your-api/verify-password';
```

```javascript
// In auth.js, update all instances:
```

## TROUBLESHOOTING

### Issue: Infinite redirect loop
**Solution**: Ensure login page is excluded from auth check (`isLoginPage` check)

### Issue: Can't access pages after login
**Solution**: Check localStorage keys match between auth.js and login.html

### Issue: Password verification fails
**Solution**: 
- Check server endpoint returns `{ success: true }` format
- Verify CORS settings on server
- Check network tab in browser dev tools

### Issue: Redirect doesn't preserve URL
**Solution**: Ensure `redirectToLogin()` properly encodes the current URL

### Issue: Doesn't work in subdirectories
**Solution**: Verify `getLoginPagePath()` calculates relative paths correctly

## SUMMARY FOR AI AGENT IMPLEMENTATION

To implement this password protection system on a new webpage:

1. **Create server endpoint** that accepts POST requests to `/api/verify-password` with `{ password: string }` and returns `{ success: boolean }`

2. **Create auth.js** with:
   - Configuration constants (token keys, session duration, API endpoint)
   - Authentication check functions (`isAuthenticated`, `setAuthenticated`, `clearAuthentication`)
   - Server communication (`verifyPassword`)
   - Path calculation functions for relative paths
   - Main protection logic that runs immediately on script load

3. **Create login.html** with:
   - Password input form
   - Form submission handler that calls `window.auth.checkPassword()`
   - Success handler that calls `window.auth.setAuthenticated()` and redirects
   - Error display for failed attempts

4. **Include auth.js in `<head>`** of all protected pages (before page content loads)

5. **Test** authentication flow, redirects, session persistence, and logout

The system uses localStorage for token storage and requires the auth.js script to run synchronously before page content renders to effectively protect pages.
